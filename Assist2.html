Problemstellung & Ansatz

Typische Pain Points

  • Viele Assistenten verarbeiten Sprache (STT) und Antworten (LLM) vollständig in der Cloud
  • Datenschutz: Je mehr Cloud-Anteile, desto mehr Daten verlassen das Gerät
  • Latenz variiert stark (Netzwerk/Provider) – Cloud-Schritte erhöhen Abhängigkeit von Roundtrips
  • Abhängigkeit von Cloud-APIs (z. B. LLM/TTS) und deren Limits/Verfügbarkeit

Implementierter Ansatz

  • Hybrid: lokale Erkennung (Vosk) + Antwortgenerierung via Gemini (Cloud)
  • Datenschutz klar getrennt: Audio bleibt lokal; erkannter Text wird für Gemini gesendet
  • Multithreading: Audioaufnahme, LLM und TTS laufen parallel und entkoppelt
  • Wake/Sleep: Aktivierung per Wake-Phrase, Deaktivierung per Sleep-Phrase

Kernidee: Session-Verwaltung per State + session_id (Thread-sicher über Lock)

Systemarchitektur

Resilienz & Error Handling

Retry-Strategie (Exponential Backoff)

Attempt 1 0.6s + jitter
Attempt 2 1.2s + jitter
Attempt 3 2.4s + jitter
Max: 5 Versuche, Max Sleep: 8s

Fehlerkategorien

Retry 429 (Rate Limit), 5xx, Timeout, Netzwerkprobleme
Stop Quota Exhausted / RESOURCE_EXHAUSTED → Abbruch/Leeren der Queue
Queue Queue.Full → Flush & Weiter
error_handling.py (Auszug) 
def should_retry(exc: Exception) -> bool:
    status = getattr(exc, "status_code", None)
    if status == 429 and _looks_like_quota_exhausted(exc):
        return False  # quota: nicht retrybar
    if status in (429, 500, 502, 503, 504):
        return True
    return "timeout" in str(exc).lower()
